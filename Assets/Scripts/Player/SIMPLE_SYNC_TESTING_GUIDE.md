# Руководство по тестированию SimplePlayerSync для FishNet 4

## Что было сделано

Создана новая система синхронизации игроков `SimplePlayerSync_FishNet4` для решения проблемы "скольжения" клиентов при коллизиях с объектами. Система полностью совместима с FishNet 4 и использует только RPC вместо устаревших SyncVar.

## ⚠️ Важно: Совместимость с FishNet 4

**Проблема**: В FishNet 4 убрали поддержку `SyncVar` и `SyncObject`, поэтому первоначальная версия не работала.

**Решение**: Новая версия `SimplePlayerSync_FishNet4` использует только:
- `[ServerRpc]` - для отправки данных от клиента на сервер
- `[ObserversRpc]` - для рассылки данных от сервера к наблюдателям  
- Никаких `SyncVar` или других устаревших атрибутов

### Созданные файлы:
- `SimplePlayerSync_FishNet4.cs` - Основная система синхронизации для FishNet 4
- `PlayerPrefabUpdater.cs` - Утилиты для обновления префаба
- `SIMPLE_SYNC_TESTING_GUIDE.md` - Данное руководство

### Измененные файлы:
- `PlayerController.cs` - Добавлена поддержка новой системы синхронизации

## Шаги для внедрения и тестирования

### 1. Обновление префаба игрока

В Unity Editor выполните следующие действия:

1. Откройте меню **Tools → Player Sync → Show Current Sync Status**
   - Это покажет текущее состояние синхронизации

2. Выберите **Tools → Player Sync → Update Player Prefab for SimplePlayerSync**
   - Это автоматически:
     - Отключит NetworkTransform (но не удалит)
     - Добавит SimplePlayerSync_FishNet4 с настройками по умолчанию
     - Включит отладочные логи для тестирования

3. Проверьте префаб `Assets/Prefab/Player.prefab`:
   - NetworkTransform должен быть отключен (enabled = false)
   - SimplePlayerSync_FishNet4 должен быть добавлен и включен

### 2. Настройка SimplePlayerSync_FishNet4

Откройте префаб Player и настройте параметры SimplePlayerSync_FishNet4:

```
Position Threshold: 0.1f      // Минимальное расстояние для синхронизации
Rotation Threshold: 5f        // Минимальный угол поворота для синхронизации
Lerp Speed: 15f              // Скорость интерполяции для удаленных игроков
Teleport Distance: 5f        // Расстояние для телепортации вместо интерполяции
Force Sync Interval: 1f      // Интервал принудительной синхронизации (сек)
Enable Debug Logs: true      // Включить отладочные логи
```

### 3. Тестирование

#### Тест 1: Базовая синхронизация
1. Запустите сервер (хост)
2. Подключите клиент
3. Проверьте консоль на наличие логов:
   ```
   [SimplePlayerSync_FishNet4] Owner initialized at (x, y, z)
   [SimplePlayerSync_FishNet4] Remote player initialized at (x, y, z)
   ```

#### Тест 2: Проблема "скольжения"
1. На клиенте подойдите к стене или объекту
2. Попытайтесь "упереться" в объект и двигаться
3. Наблюдайте на хосте - игрок НЕ должен скользить
4. Проверьте логи синхронизации:
   ```
   [SimplePlayerSync_FishNet4] Owner sent update: pos=(x,y,z), rot=(x,y,z)
   [SimplePlayerSync_FishNet4] Remote received update: target=(x,y,z)
   ```

#### Тест 3: Телепортация и принудительное движение
1. Протестируйте лассо и веревки
2. Проверьте, что телепортация работает корректно
3. Убедитесь, что нет конфликтов с принудительным движением

### 4. Отладка

#### Полезные логи в консоли:
- `[SimplePlayerSync_FishNet4]` - логи новой системы синхронизации
- `[RopeDbg SimpleSync_FishNet4]` - логи принудительного движения с новой системой
- `[PlayerController]` - логи PlayerController с поддержкой новой системы

#### Параметры для настройки при проблемах:

**Если игроки "телепортируются":**
- Уменьшите `Teleport Distance` до 2-3f
- Увеличьте `Lerp Speed` до 20-25f

**Если синхронизация слишком частая:**
- Увеличьте `Position Threshold` до 0.2f
- Увеличьте `Force Sync Interval` до 2f

**Если движение кажется "рывками":**
- Уменьшите `Position Threshold` до 0.05f
- Увеличьте `Lerp Speed` до 30f

### 5. Возврат к старой системе (если нужно)

Если новая система работает неудовлетворительно:

1. Выберите **Tools → Player Sync → Revert to NetworkTransform**
2. Это вернет систему к использованию NetworkTransform

## Технические детали

### Принцип работы SimplePlayerSync_FishNet4:
1. **Владелец** рассчитывает движение через PlayerPhysics
2. **Отправляет позицию** только при значимых изменениях (threshold)
3. **Сервер валидирует** движение (защита от читов)
4. **Удаленные копии** получают позицию и плавно интерполируют

### Отличия от NetworkTransform:
- Нет конфликтов с Rigidbody
- Более гибкая настройка чувствительности
- Простая интеграция с существующим кодом
- Меньше сетевого трафика при правильной настройке

## Поддержка и отладка

При возникновении проблем:

1. Включите `Enable Debug Logs` в SimplePlayerSync_FishNet4
2. Проверьте консоль Unity на предмет ошибок
3. Используйте **Tools → Player Sync → Show Current Sync Status** для диагностики
4. При необходимости вернитесь к NetworkTransform для сравнения
